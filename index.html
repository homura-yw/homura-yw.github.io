GCC编译器的安装与使用

C\C++ 2022/8/24

缘由

由于电脑内存有点不够用了，所以就把visual studio 2022卸载了，但需要编写c++的时候，还是需要一个编译器的，vs这种航母级的IDE对于我目前的水平来说，还是过于庞大，很多功能完全用不上，故决定下载安装一个GCC编译器来编译我自己写的C++代码。

GCC简介

以下摘自百度百科

GCC（GNU Compiler Collection，GNU编译器套件）是由GNU开发的编程语言译器。GNU编译器套件包括C、C++、 Objective-C、 Fortran、Java、Ada和Go语言前端，也包括了这些语言的库（如libstdc++，libgcj等。） GCC的初衷是为GNU操作系统专门编写的一款编译器。GNU系统是彻底的自由软件。此处，“自由”的含义是它尊重用户的自由。

GCC编译器的使用

由于这一段我不是很熟悉，如果有出错，可以通过about中的联系方式来指出我的问题。

C/C++的编译步骤分为

预处理->编译->汇编->链接

预处理

预处理是将所有的.c,.cpp扩展名的文件转化为.i扩展名的文件。

预处理的目的是处理程序中的预处理命令，预处理命令均是以“#”开头的语句，如：

  • include
  • define
  • ifdef
  • ifndef

此外，还会自动删除程序中的注释，当然，这自不比多说。

可在终端输入命令

gcc -E hello.c -o hello.i

将其转化为预编译文件。这里可以做个示范，我们写一个简单的输出”hello world”的c语言程序

//文件名为hello.c
#include <stdio.h>

int main()
{
    printf("hello world");
    return 0;
}

预编译后的程序是这样的

编译

编译就是把C/C++的预编译文件转变为汇编语言代码，其扩展名为.s

在命令行内输入

gcc -S hello.i -o hello.s

之后我们的代码生成的汇编代码是这样的

    .file    "hello.c"

汇编

到了汇编这一步，就是将我们的汇编代码转变为二进制机器码文件。

汇编就是将第二步输出的汇编代码翻译成符合一定格式的机器代码，在Linux系统上一般表现为ELF目标文件(OBJ文件)。“反汇编”是指将机器代码转换为汇编代码，这在调试程序时常常用到。

在命令行输入命令

gcc -c hello.s -o hello.o

链接

链接这一步我就不是很熟悉了，直接摘要csdn上的一些博客原文

链接就是将上步生成的OBJ文件和系统库的OBJ文件、库文件链接起来，最终生成了可以在特定平台运行的可执行文件。

其中，Windows系统的可执行文件就是扩展名为.exe的文件。

在命令行中输入

gcc hello.o -o <filename>

即可生成可执行文件

总结

有没有一种方式，可以直接将c/c++文件一步到位呢？其实很简单，直接输入命令

gcc hello.c -o <filename>

就直接将源文件转化为可执行文件。但是这样做，包括之前一步一步来的做法，都有一个坏处，就是当源文件出问题时，编译不会成功，也不会有任何提示，要解决办法也很简单，将上个命令替换为

gcc -Wall hello.c -o <filename>

这个Wall不是指墙，而是指W(Warning)+all(进行所有的流程),当然不必过于纠结这个问题。
那假如程序有问题，提示是怎样的呢？例如我们直接将程序修改一下：

#include <stdio.h>

int main()
{
    printf("hello world",4);
    return 0;
}

命令总结

选项 功能
-v 查看编译器的版本
-o 指定输出文件名
-E 只预编译，不会编译，汇编，链接
-S 只运行到编译，不会汇编，链接
-c 编译和汇编,不会链接

由一道算法题所启发的井字棋游戏

算法 2022/8/23

缘由

  • 几个月前参加学校举办的算法比赛时，遇见过一个难度颇为不易的题目，题目大致内容是通过输入一个井字棋的棋盘，然后求某方要赢的情况的种数。虽然井字棋本身非常的简单，但要用程序模拟出下棋的情况总的种数，终归还是难到我了。虽然但是，我通过不断递归遍历的算法，终究还是把这道题做出来了。
  • 这道题用递归能适用，很大一部分在于，井字棋本身是非常简单的，总共不过3*3=9个棋子，考虑到棋盘上已经有棋子了，所以每种情况递归的层数都不会超出8，这些为递归提供了条件。
  • 将这道题水过去后，我一直感觉，本题在某种程度上几乎就是让我写了一个预测所有情况的AI，将所有的情况综合考虑，即可得出，在走 剩余格子 中的一个后，所获胜的种数，所失败的种数以及平局的种数，进而可以求得走这一步获胜的概率！而只要每一步都走获胜概率最高的一步，这不就形成了一个可以自己下棋的AI了吗？理论成立，实践的种子也埋在了我的心里，也正是这样促使我完成一个简单的井字棋游戏。

游戏AI内核的模块

在上面，有一个非常关键的函数

void putPiece(char qipan[][3])

这个函数的作用便是，传入棋盘的数组后，将决定走下一步棋的位置，让我们看看在”井.h”中的putPiece函数是怎么写的：

void putPiece(char qipan[][3]) {
    void Copy(char qipan[][3], char Qipan[][3]);
    int stepin(char qipan[][3], int cnt, int step, int W);

    int cnt = 0;
    for (int I = 0; I < 3; I++) {
        for (int i = 0; i < 3; i++) {
            if (qipan[I][i] == '_') cnt++;
        }
    }
    char Qipan[3][3] = { 0 };
    double gailv[9] = { 0 };
    int top = 0;
    for (int I = 0; I < 3; I++) {
        for (int i = 0; i < 3; i++) {
            if (qipan[I][i] == '_') {
                Copy(qipan, Qipan);
                Qipan[I][i] = 'O';
                double X_win = stepin(Qipan, cnt - 1, 1, 1);
                double O_win = stepin(Qipan, cnt - 1, 1, 2);
                double ping = stepin(Qipan, cnt - 1, 1, 0);
                gailv[top++] = (ping + O_win) / (ping + O_win + X_win);
            }
        }
    }
    double max = 0;
    int num = 0;
    for (int i = 0; i < top; i++) {
        if (gailv[i] >= max) {
            max = gailv[i];
            num = i;
        }
    }
    int count = 0;
    for (int I = 0; I < 3; I++) {
        for (int i = 0; i < 3; i++) {
            if (qipan[I][i] == '_') {
                count++;
            }
            if (count == num + 1) {
                qipan[I][i] = 'O';
                return;
            }
        }
    }
}

这其中有两个函数

void Copy(char qipan[][3], char Qipan[][3]);

int stepin(char qipan[][3], int cnt, int step, int W);

其中Copy结合上下文不难理解，这个其实就是一个简单的复制棋盘数组的代码
真正重要的函数便是这个stepin，这也是整个程序不断递归的函数（同时参数也多得离谱）
stepin函数的作用，就是返回对应结局的种数，这里先贴一下代码，不然之后讲起来就有点混乱,因为这个函数还是有点复杂的

int stepin(char qipan[][3], int cnt, int step, int W) {
    bool isnotInline(char qipan[][3]);
    int whowin(char qipan[][3]);
    void Copy(char qipan[][3], char Qipan[][3]);
    void set(char* site, int step);

    if (whowin(qipan) != 0) {
        if (whowin(qipan) == W) return 1;
        else return 0;
    }

    if (cnt == 1) {

        int right = 1;
        int wrong = 0;

        char Qipan[3][3];
        Copy(qipan, Qipan);
        char* p;
        for (int I = 0; I < 3; I++) {

            for (int i = 0; i < 3; i++) {
                if (Qipan[I][i] == '_') {
                    p = &Qipan[I][i];
                    set(p, step);
                    goto end;
                }
            }
        }
    end:
        if (W == whowin(Qipan)) {

            return right;

        }
        else return wrong;

    }

    

    int sum = 0;
    for (int J = 0; J < cnt; J++) {
        char QiPan[3][3];
        Copy(qipan, QiPan);

        int j = 0;
        char* p;
        for (int I = 0; I < 3; I++) {

            for (int i = 0; i < 3; i++) {
                if (QiPan[I][i] == '_') {
                    p = &QiPan[I][i];
                    j++;
                    if (j > J) {
                        set(p, step);
                        goto END;
                    }
                }
            }
        }
    END:
        sum += stepin(QiPan, cnt - 1, step + 1, W);
    }
    return sum;
}

这个函数中，参数cnt代表这个棋盘还剩下的棋子数，step代表当前已走的棋子数（有一说一，这俩其实知道一个就行了，主要是这是全盘抄的比赛时的代码，当时没有多想，现在又不敢乱动这个代码了，因为调试起来真的会死人），至于这个W，则是选定的要赢的棋子（或者更妥当一点，应该是选择X或O赢或平局）。其中还得简要提一嘴这个函数

int whowin(char qipan[][3])

这个就是判断棋子胜负的函数，返回1代表X赢，返回2代表O赢，返回0代表平局或者棋盘未满。当

if (whowin(qipan) == W)

成立时，stepin停止递归，并返回1，当不成立且w!=0时，返回0

对stepin函数的总结

stepin函数继续递归的条件:

  • 无任意一方棋子连为3个

  • cnt值不为0
    其中，最后一个for循环就是对于走每一个空格的满足参数W情况的平行递归的总和，即：

    END:
      sum += stepin(QiPan, cnt - 1, step + 1, W);
